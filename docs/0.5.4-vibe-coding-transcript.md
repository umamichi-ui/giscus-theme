## 助手 (3)

根因是：**两边的 rem 参照系和行高在移动端不一致，桌面端被 giscus 主题的根字号放大「碰巧对齐」了。**

### 1. 文章标签：全站 rem 基准不变

`umamichi.moe` 的 `.article-tag`（`src/styles/site-layout.css`）：

- `padding: 0.25rem 0.75rem`
- `font-size: 0.875rem`
- `border-radius: 9999px`

站点在 `≥48rem` 时只改 CSS 变量 `--site-content-font-size`（`.site-route-main` 用），**没有**改 `html { font-size }`。`.article-tag` 的 rem 始终相对文档根（通常 16px），**手机与桌面物理尺寸相同**。标签在 `.site-route-main` 内，继承 `line-height: 1.65`。

### 2. Giscus 反应区：iframe 根字号会随断点变

giscus 主题 `_umamichi-type.css` 在 iframe 内直接改 `html font-size`：

- 默认 `1rem`
- `≥48rem` → `1.125rem`（镜像正文 `--site-content-font-size`）

反应胶囊 `_umamichi-giscus-ui.css` 用 rem，并设：

- `--gsc-reaction-pill-height: calc(0.5rem + 1.225rem)`（按 `line-height: 1.4` 估算）
- `padding: 0.25rem 0.75rem`
- `font-size: 0.875rem`
- `line-height: 1.4 !important`

### 3. 高度估算（根 16px）

| | 文章标签 | Giscus 反应胶囊 |
|---|---|---|
| 上下 padding | 8px | 8px |
| 字号 | 14px | 14px |
| 行高 | **1.65** | **1.4** |
| 总高约 | **≈31px** | **≈27.6px** |

移动端两边 rem 根都是 16px，但 giscus 行高更小、`min-height` 按 1.4 算，故比标签矮约 3–4px。

桌面端 iframe 根变为 18px：`1.725rem × 18 ≈ 31px`，与标签 ≈31px **碰巧对齐**；标签本身不随断点放大。

### 4. 小结

| 因素 | 文章标签 | Giscus 反应区 |
|------|----------|---------------|
| rem 参照 | 始终文档根 16px | 移动 16px，桌面 18px |
| 是否随 48rem 放大 | 否 | 是 |
| 行高 | 1.65 | 1.4 |

## 助手 (6)

已在 `@umamichi-ui/giscus-theme` **0.5.4** 改好。核心思路：**反应区尺寸镜像 `.article-tag` 的 document-root rem，并用 `--gsc-site-root-rem-factor` 抵消 iframe 在 `≥48rem` 时把 `html` 放大到 `1.125rem` 的影响**。

### 改动

**`_umamichi-scale-tokens.css`** — 新增 token：

- `--gsc-site-root-rem-factor`：移动 `1`，桌面 `1.125`
- `--gsc-article-tag-font-size` / `padding-block` / `padding-inline` / `line-height: 1.65` / `gap`
- `--gsc-reaction-pill-height`: `calc((0.5rem + 0.875rem × 1.65) / factor)`

**`_umamichi-giscus-ui.css`** — 反应胶囊、笑脸圆钮、upvote、emoji 图标改用上述 token；行高 `1.4` → `1.65`；图标 `1rem` → `1em`。

已 `npm run build`，8 个主题 CSS 重新生成。

### 效果

| 视口 | 之前 | 现在 |
|------|------|------|
| 移动 | 反应区 ≈27.6px，标签 ≈31px | 均 ≈31px |
| 桌面 | 碰巧接近 | 仍 ≈31px（不再随 iframe 根字号变大） |
