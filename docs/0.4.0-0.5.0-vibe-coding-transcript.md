## 建议修改位置

在 giscus-theme 仓库中，主要改两个源文件（构建后合并进 `styles/themes/*.css`）：

### 1. `_umamichi-aqua-light.css` / `_umamichi-aqua-dark.css`（及对应 palette 覆盖层）

**第 2 点：框内灰色 → 背景色**

把 Primer 画布映射从灰色改为背景色：

```css
/* 改前 */
--color-canvas-subtle: var(--gray-100);
--color-canvas-inset: var(--gray-100);

/* 改后 */
--color-canvas-subtle: var(--site-bg);
--color-canvas-inset: var(--site-bg);
```

深色主题同理，用 `var(--site-bg)`，不要用 `--site-surface-muted` / `--gray-800`，否则输入区内仍会偏灰。

### 2. `_umamichi-chrome.css`

**拆分当前的「全部直角」规则**

现在末尾整块把 tab、按钮、输入区都设为 `border-radius: 0`。应改为：

- **保留直角**：评论卡片、回复列表等大块表面  
  （`.gsc-comment > div`、`.gsc-comment-content`、`.gsc-replies`、头像 `.rounded-xl` 等）
- **从直角列表中移除**：  
  `.gsc-comment-box-tabs button`、`.gsc-comment-box-write`、`.gsc-comment-box-textarea`、`.gsc-reply-box button`（若需与主按钮一致）

**第 1 点：文本框样式（对齐 njmetro / forms.css）**

giscus 的可视「框」是 `.gsc-comment-box-write`（内部是 `.gsc-comment-box-textarea`）。建议用 `:focus-within` 模拟 common-css 的 focus ring：

```css
.gsc-comment-box-write {
  border: 1px solid var(--site-border) !important;
  border-radius: var(--radius-sm) !important;
  background: var(--site-bg) !important;
  outline: none;
  transition: border-color var(--transition-fast);
}

.gsc-comment-box-write:hover {
  border-color: var(--site-border-strong) !important;
}

.gsc-comment-box-write:focus-within {
  border-color: var(--site-accent) !important;
  outline: 2px solid var(--site-focus-ring) !important;
  outline-offset: 1px;
  border-radius: var(--radius-sm) !important;
}

.gsc-comment-box-textarea {
  background: transparent !important;
  border-radius: var(--radius-sm) !important;
}
```

预览区若也有灰底，可同样处理 `.gsc-comment-box-preview`。

**第 3 点：按钮、选项卡圆角**

```css
.gsc-button,
.gsc-comment-box-tabs button,
.gsc-reply-box .gsc-button {
  border-radius: var(--radius-sm) !important;
}
```

排序 segmented control（「最早 / 最新」）若仍有 Tailwind 圆角或被其他规则覆盖，可补充：

```css
.gsc-left-header button {
  border-radius: var(--radius-sm) !important;
}
```

## 注意事项

1. **特异性**：现有规则带 `!important`，新规则也需 `!important`，或从旧列表中移除对应选择器。
2. **`:focus-within` vs `:focus-visible`**：giscus 输入区是复合容器（含 Markdown 工具栏），用 `:focus-within` 比直接给 textarea 加 `:focus-visible` 更接近 common-css 的「整框高亮」效果。
3. **评论卡片仍可为直角**：你只提了文本框、按钮、选项卡；0.2.2 的「评论表面直角」可保留。
4. **构建与发布流程**：
   - 在 giscus-theme 仓库：`npm run build`
   - 发布新版本（如 `0.3.3`）
   - 在 `umamichi.moe`：升级依赖并执行 `npm run sync:giscus-theme`，将主题同步到 `public/giscus/`

## 建议修改（`_umamichi-chrome.css`）


- 从「全局直角」列表里 **移除** `.gsc-comment-box-tabs button`、`.gsc-comment-box-write`、`.gsc-comment-box-textarea`、`.gsc-comment-box-textarea-extras`

### 3：操作按钮圆角对齐 njmetro

njmetro 弹窗按钮用的是 common-css 的 `.primary-button` / `.secondary-button`，圆角为 **`--radius-sm`（10px）**：

```17:17:C:\Users\Umamichi\Desktop\umamichi-ui-common-css\styles\primitives.css
    border-radius: var(--radius-sm);
```

应 **只** 给操作按钮设圆角，不要把 tab、排序 segmented control 混在同一规则里：

```css
/* 从现有规则中移除 .gsc-comment-box-tabs button、.gsc-left-header button */

```

若「取消」视觉上仍不像 `secondary-button`，需在 giscus iframe 里确认其 class（通常也是 `.gsc-button`，但可能是非 primary 样式）；可再补：

```css
.gsc-reply-box .gsc-button:not(.btn-primary) {
  background: var(--site-button-bg);
  color: var(--site-button-fg);
}
```

主按钮（评论 / 回复 / 登录）已在 `_umamichi-aqua-*.css` 里映射 `--site-button-accent-*`，与 `.primary-button` 一致。

### 4：光晕颜色对齐 njmetro 生成选项文本框

njmetro 生成选项使用 `.text-input`，focus 样式来自 common-css：

```63:70:C:\Users\Umamichi\Desktop\umamichi-ui-common-css\styles\forms.css
  .text-input:focus-visible {
    border-color: var(--site-accent);
    outline: 2px solid var(--site-focus-ring);
    outline-offset: 1px;
  }
```

`--site-focus-ring` 在浅色为 `color-mix(in srgb, var(--site-accent) 22%, transparent)`（`tokens.css` 第 56 行）。

当前光晕可能偏色/偏粗，常见原因：
1. giscus / Tailwind 在 textarea 上额外加了 `box-shadow` 或 `ring-*`
2. focus 环画在整块 `.gsc-comment-box-write` 上，与单输入框视觉不同

建议：

```css
.gsc-comment-box-write:focus-within {
  border-color: var(--site-accent) !important;
  outline: 2px solid var(--site-focus-ring) !important;
  outline-offset: 1px;
  box-shadow: none !important;
}

.gsc-comment-box-textarea,
.gsc-comment-box-textarea:focus,
.gsc-comment-box-textarea:focus-visible {
  outline: none !important;
  box-shadow: none !important;
}
```

这样光晕只由 `--site-focus-ring` 控制，与 njmetro 的 `.text-input` 一致；无需改 token 定义。

## 0.4.4 为什么看起来怪

当前 DOM（giscus）是：

```
.gsc-comment-box-tabs          ← Write / Preview
.gsc-comment-box-main
  .gsc-comment-box-write       ← 0.4.4 在这里加了边框 + :focus-within 光晕

.gsc-comment-box-bottom        ← 评论按钮（在外层）
```

0.4.4 实际叠了三层边框：

| 层 | 样式 | 你看到的效果 |
|----|------|----------------|
| 外层 | `.gsc-comment-box-write` 灰线 + `outline` 光晕 | 多出来一圈带光晕的矩形 |
| 内层上 | `textarea` 自带 `border` + 上圆角 | 「真正的」输入区 |
| 内层下 | `extras` 自带 `border` + 下圆角 | 工具条 |

所以会出现：**外面一圈带光晕的方框，里面再套一个圆角框**。

## 重做时应遵守的规格（单层模型）

目标结构：

```
[Write][Preview]     ← tab 仅上圆角，无输入框光晕
┌──────────────────────────────────── 编辑矩形：一层边框、直角 ────┐
│ ╭──────── textarea ────────╮                                      │
│ │                          │  ← 内缘视觉圆角用 overflow 实现，     │
│ ├──────────────────────────┤     不要再给 textarea/extras 各画一圈 border
│ │  Markdown 工具条          │                                      │
│ ╰──────────────────────────╯                                      │
└───────────────────────────────────────────────────────────────────┘
[退出登录]  [评论]    ← 在编辑矩形外
```

实现要点：

1. **`.gsc-comment-box-write`：唯一可见边框 + 唯一 `:focus-within` 光晕**
   - `border-radius: 0`（外轮廓直角）
   - `overflow: hidden`（用内裁切做出上下圆角，而不是子元素再画框）
   - **不要** `padding: 6px` + 子元素第二圈 border

2. **`textarea` / `extras`：去掉独立 `border`**
   - 只保留 `textarea { border-bottom: 1px solid }` 作分隔线
   - 去掉 `form-control` 自带边框：`border: none; box-shadow: none`

3. **圆角只在一处定义**（二选一，不要混用）  
   - 要么 `write` 上 `border-radius: 0` + 内层 `background` 用 `border-radius` + `overflow: hidden`  
   - 要么整框 `write` 用 `0 0 var(--radius-sm) var(--radius-sm)`（若你接受编辑区下沿圆角），**但外层仍不要第二层光晕**

4. **光晕不要挂在外层 padding 壳上**  
   - 删除「外壳有 border + 内层又有 border」的模式  
   - `:focus-within` 只改 **一条** 边框颜色 + **一条** `outline`（对齐 njmetro `forms.css`）

5. **继续保留 0.4.3 的**  
   - 不对 `.gsc-comment-box`（form）整体加输入框样式  
   - `.gsc-comment-box-bottom` 按钮圆角覆盖 `.btn` / `.rounded-md`  
   - `.gsc-reply-box > button` 折叠「写回复」样式

## 原因

giscus 用 Primer 工具类把**背景色绑在 CSS 变量**上（见上游 `styles/base.css`）：

| 工具类 | 变量 | 你们主题里当前映射 |
|--------|------|-------------------|
| `color-bg-primary` / `color-bg-default` | `--color-canvas-default` | `--site-bg`（页底） |
| `color-bg-secondary` / `color-bg-subtle` | `--color-canvas-subtle` | `--site-surface-muted`（灰底） |
| `color-bg-inset` | `--color-canvas-inset` | `--site-surface-muted`（灰底） |

**评论卡片**外层通常是 `color-bg-secondary`（或类似 subtle 类）→ 会吃到 `--site-surface-muted`，所以是灰底。

**评论编辑区** `<form class="color-bg-primary … gsc-comment-box">` 理论上是 `--site-bg`；你们又在 `_umamichi-chrome.css` 里给 textarea / 预览等写了 `background: var(--site-bg) !important`。若仍见灰，多半是：

- 父级/兄弟节点用了 `color-bg-secondary` / `color-bg-tertiary`
- tab 区、`.gsc-comment-box-main` 等未单独覆盖
- 嵌在评论卡片里的回复编辑器继承了卡片灰底

**不能**再把全局 `--color-canvas-subtle` 改回 `--site-bg`——0.4.7 刚用它恢复**代码块**灰底，一改又会让 `pre` / 行内代码变白。

```css
/* 评论卡片 + 编辑外壳：与页面同色，不用 canvas-subtle 灰底 */

.gsc-comment-box,
.gsc-comment-box-tabs,
.gsc-comment-box-main,
.gsc-comment-box-bottom,
.gsc-comment-content,
.gsc-replies,

/* 嵌在评论里的回复编辑框 */

.gsc-replies .gsc-comment-box {

}
```

- 选的是 **giscus 自己的 `gsc-*` 结构**，不会误伤 `.highlight pre`、`.markdown pre`（它们在 `.gsc-comment-content` 内部，且代码块背景来自 `canvas-inset` / `color-bg-inset`，需要单独保留）

```css
.gsc-comment-box-preview.markdown pre,
.gsc-comment-content.markdown :not(pre) > code,
.gsc-reply-content.markdown :not(pre) > code {
```

## 2. 内联的 common-css token（构建进 `light.css` / `dark.css`）

这些在 `tokens.css` 里是 **px**，不随 `html { font-size: 1.125rem }` 变：

| 变量 | 当前 | 影响 |
|------|------|------|
| `--radius-sm/md/lg` | `10px` / `14px` / `20px` | 圆角、编辑框 `::after`、按钮 |
| `--control-height` | `38px` | 「写回复」触发器高度 |
| `--control-padding-inline` | `12px` | 输入框、回复按钮左右内边距 |

**建议**：在 `styles/src/` 加一小段（例如 `_umamichi-scale-tokens.css`，在 entry 里放在 `tokens` 之后）：

```css
:host,
:root {
  --radius-sm: 0.625rem;
  --radius-md: 0.875rem;
  --radius-lg: 1.25rem;
  --control-height: 2.375rem;
  --control-padding-inline: 0.75rem;
}
```

这样所有 `var(--radius-sm)`、`var(--control-height)` 会随根字号缩放。
